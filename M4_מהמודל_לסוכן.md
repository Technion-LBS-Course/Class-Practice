## הלולאה — ארבעה שלבים

1. היוזר כותב בשפה חופשית.
2. ה-LLM מחלץ מהטקסט את הפרמטרים שהמודל צריך (כ-JSON).
3. **המודל שלכם** מריץ `predict` על הפרמטרים.
4. מציגים ליוזר את התוצאה (טקסט פשוט, או ניסוח של ה-LLM).

שלב 3 הוא הלב. אם הסוכן שלכם נעצר בשלב 2 — חילצתם פרמטרים ולא קראתם למודל — בניתם מחלץ טקסט, לא מוצר חיזוי.

## תבנית 4 השדות — תתחילו מפה

ארבע שורות שמגדירות את הסוכן שלכם. הן גם **מסדרות את הפרויקט**: מי שמתקשה להגדיר מה הוא בונה, השאלה "מה היוזר נותן ומה המודל צריך" פותרת לו את זה.

- היוזר אומר: ______ (טקסט חופשי)
- הפרמטרים שאני מחלץ: ______
- הקלט שהמודל שלי צריך: ______
- מה שחוזר ליוזר: ______ (הפלט של המודל, מנוסח ביושר)

> 🟦 רגרסיה: היוזר כותב "דירה 3 חדרים בתל אביב, 80 מטר" ← מחלצים `{rooms, city, area}` ← המודל מחזיר מחיר ← "המחיר המשוער: 2.4M".
> 🟩 סיווג / סגמנטציה: היוזר מתאר מקום ← מחלצים את הפיצ'רים ← המודל מחזיר קטגוריה או סגמנט ← "המקום הזה שייך לסגמנט X".
> 🟨 זיהוי (קלט תמונה): כאן הקלט הוא **תמונה**, לא טקסט. הסוכן עוטף קריאה ל-API חיצוני (מיקום / מזג אוויר) ואת הרצת הגלאי על התמונה, ומחזיר התראה מנוסחת.

---

## פרומפט: system מול user

- ה-system: כותבים אותו **פעם אחת**. מי הסוכן, מה מותר ואסור לו, ובאיזה פורמט להחזיר (JSON עם שדות מוגדרים). זה הגבול — היוזר לא יכול לדרוס אותו.
- ה-user: **משתנה בכל פנייה**. זה הטקסט החופשי של היוזר.

החוקים שחשוב לזכור: ה-LLM הוא **stateless** (שולחים לו את כל ההקשר בכל קריאה), `temperature` נמוך כשרוצים JSON אמין, והמפתח נשמר ב-`st.secrets` ולא נכנס לעולם לקוד או לגיט.

---

## שלושת הרכיבים של M4

1. ולידציית קלט — לבדוק שמה שחולץ הגיוני (שדות לא חסרים, ערכים בטווח).
2. הנדסת prompt — system ברור עם פורמט פלט מוגדר.
3. ולידציית פלט + Fallback — אם ה-LLM החזיר משהו לא תקין, לתפוס את זה ולתת ברירת מחדל. LLM יכול להחזיר טקסט במקום JSON או להמציא — זה מובנה, ולכן ה-Fallback הוא חובה.

## המימוש: Groq + Llama (חינם)

מקבלים מפתח חינם ב-groq.com ומריצים מודלים פתוחים בחינם (llama / gpt-oss). שימו לב: **Groq תואם-OpenAI, לא Anthropic** — הקריאה היא `client.chat.completions.create`, לא `client.messages.create`.

```python
# pip install groq    (Groq is OpenAI-compatible, NOT Anthropic)
import json, pandas as pd, streamlit as st
from groq import Groq

client = Groq(api_key=st.secrets["GROQ_API_KEY"])   # never hard-code the key, never commit it

SYSTEM = (
    "You extract parameters from the user's free text. "
    "Return ONLY JSON with the keys: area, rooms, city. No extra text."
)

def extract_params(user_text: str) -> dict:
    resp = client.chat.completions.create(
        model="llama-3.3-70b-versatile",   # or openai/gpt-oss-20b — pick from Groq's list
        temperature=0,                     # low temperature for reliable JSON
        messages=[
            {"role": "system", "content": SYSTEM},
            {"role": "user",   "content": user_text},
        ],
    )
    try:
        return json.loads(resp.choices[0].message.content)   # output validation
    except json.JSONDecodeError:
        return {"area": 80, "rooms": 3, "city": "תל אביב"}    # Fallback

# --- the full loop ---
user_text = st.text_input("מה תרצו לחזות?")
if user_text:
    params = extract_params(user_text)              # steps 1-2: LLM extracts params
    y = pipe.predict(pd.DataFrame([params]))[0]     # step 3: YOUR model predicts
    st.success(f"תוצאה: {y:.1f}")                    # step 4: present the model's number
```
